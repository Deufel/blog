How it works:

The general idea with this layout is that each page containes an auto adjusting "hero" grid from the < body > tag

if you add a popover="auto" attribute to the body > head, aside, fooer, nav < then that section will be moved into a off screen drawer

note: the semantic section will determine the drawer direction

recommended use is to have the content of the page always be in the body/main/ and use the other sections for UI elements like settings, current page, ect

it is also recommended to use the same element and use js to add/remove the popover attribute when the screen gets to small

note: that only the content in the body/main will be printed

nested grid system each semantic element also allows you to use a nother layer of semantic header/nav/main/aside/footer elements withinit.

This starts to get a bit out of html semantic compliance here but I found this to be the easiest to work with and this covers almost all my layout needs
